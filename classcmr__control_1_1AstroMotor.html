Detailed Description

AstroMotor is a hardware interface for controlling a single motor with position or velocity commands.

It expects that it will only be passed a single joint within the HardwareInfo in the call to on_init(). This means, from a user perspective that the AstroMotor should only have a single motor's state interface, command interface, and joint under its control as defined in the URDFs and YAML controller config files.

A good controller for this is either position_controllers/JointGroupPositionController or velocity_controllers/JointGroupVelocityController.
